# Caddyfile.d — stateful-infrastructure route fragments

This directory holds Caddy route fragments for **stateful infrastructure
additions only** — extra databases, message queues, search engines, object
stores that live alongside Postgres/Redis/RabbitMQ/Manticore on the data
layer. The fragments are owned by `outpost onboard` (tier=compose), not
by humans, and they are gitignored on a per-installation basis.

**Stateless applications do NOT belong here.** They go on k3s tier and
expose themselves via Kubernetes IngressRoute under the
`<name>-apps.<ROOT_DOMAIN>` naming convention — caught by the broad
`*.<ROOT_DOMAIN>` Cloudflare Tunnel wildcard that routes to k3s Traefik.
See `examples/outpost.app.yaml.minimal.example` and
`examples/outpost.app.yaml.multiproduct.example`. `outpost onboard` rejects
`tier=k3s` outpost.app.yaml files that try to ship a Caddy fragment, and
it rejects `tier=compose` hosts ending in `-apps.<ROOT_DOMAIN>`.

## Why a fragment directory at all

The pre-v0.5 anti-pattern was hand-editing `core/compose/Caddyfile` whenever
a new stateful service was added — every consumer's infras repo diverged.
Fragments + `import` keep the main `Caddyfile` generic; per-installation
additions live in this directory; clean `git pull` of the infras repo
never needs merge resolution on infra routes.

## What a fragment looks like

```caddy
# core/compose/Caddyfile.d/elasticsearch.caddy
# Generated by `outpost onboard` — do not edit by hand.

@elasticsearch_host_0 host {$ES_HOST:es}.{$ROOT_DOMAIN}
handle @elasticsearch_host_0 {
    handle {
        reverse_proxy elasticsearch:9200
    }
}
```

Fragments use the same Caddy DSL as the main Caddyfile and inherit the
top-level `{ auto_https off; admin off; ... }` block automatically. They
must not declare a new `:80 { ... }` site block.

## Reloading

After dropping or editing a fragment, Caddy needs to re-read its config:

```bash
docker exec caddy caddy reload --config /etc/caddy/Caddyfile
```

`outpost onboard` does this automatically. `outpost off-board <name>`
removes the fragment + reloads.

## Cloudflare side

Stateful infra fragments expose top-level subdomains (`<prefix>.<ROOT_DOMAIN>`).
The broad `*.<ROOT_DOMAIN>` CF Tunnel wildcard points at k3s Traefik (for
apps named `<x>-apps.<ROOT_DOMAIN>`), so each stateful-infra subdomain
needs its OWN Public Hostname entry in the Cloudflare Dashboard pointing
at `http://caddy:80` to override the wildcard for that specific name.
CF Tunnel uses most-specific-wins matching — a literal entry like
`es.example.com` automatically takes precedence over `*.example.com`.
Reference: `core/compose/cloudflared/config.template.yml`.
